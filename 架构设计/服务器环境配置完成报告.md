## 下一步操作

### 1. 部署应用代码

从本地同步代码到服务器:

```bash
# 在本地执行
rsync -avz --exclude='.git' --exclude='node_modules' --exclude='.next' \
  --exclude='backend/.deer-flow' --exclude='backend/__pycache__' \
  /Users/steven/VaaT-Flow/ ubuntu@146.56.239.94:/srv/allo/
```

### 2. 配置模型 API Key

编辑环境变量文件:

```bash
ssh ubuntu@146.56.239.94
sudo vim /etc/allo/allo.env

# 取消注释并填写以下行:
# OPENAI_API_KEY=sk-your-key-here
# TAVILY_API_KEY=tvly-your-key-here
```

### 3. 安装依赖

```bash
ssh ubuntu@146.56.239.94

# 切换到 allo 用户
sudo -u allo -i

# 加载环境变量
source /etc/allo/allo.env

# 安装后端依赖
cd /srv/allo/backend
source ~/.local/bin/env
uv sync

# 安装前端依赖
cd /srv/allo/frontend
pnpm install
```

### 4. 运行数据库迁移

```bash
# 以 allo 用户执行
cd /srv/allo/backend
source /etc/allo/allo.env
source ~/.local/bin/env
PYTHONPATH=. uv run alembic upgrade head
```

### 5. 构建前端

```bash
cd /srv/allo/frontend
source /etc/allo/allo.env
pnpm build
```

### 6. 安装 systemd 服务

```bash
# 退出 allo 用户，回到 ubuntu
exit

# 复制 service 文件
sudo cp /srv/allo/deploy/systemd/*.service /etc/systemd/system/

# 重载 systemd
sudo systemctl daemon-reload

# 启用并启动服务
sudo systemctl enable allo-gateway allo-langgraph allo-frontend
sudo systemctl start allo-gateway allo-langgraph allo-frontend

# 检查状态
sudo systemctl status allo-gateway
sudo systemctl status allo-langgraph
sudo systemctl status allo-frontend
```

### 7. 配置 nginx

```bash
# 复制生产配置
sudo cp /srv/allo/deploy/nginx/allo-production.conf /etc/nginx/sites-available/allo

# 创建软链接
sudo ln -s /etc/nginx/sites-available/allo /etc/nginx/sites-enabled/allo

# 删除默认站点（可选）
sudo rm /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重载 nginx
sudo systemctl reload nginx
```

### 8. 验证部署
