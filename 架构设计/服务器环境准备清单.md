## 快速开始

### 在服务器上执行

```bash
# 1. 上传代码到服务器
rsync -avz --exclude='.git' --exclude='node_modules' --exclude='.next' \
  /Users/steven/VaaT-Flow/ root@YOUR_SERVER_IP:/srv/allo/

# 2. SSH 登录服务器
ssh root@YOUR_SERVER_IP

# 3. 运行环境初始化脚本
cd /srv/allo
bash scripts/server-bootstrap.sh

# 4. 配置 PostgreSQL（按提示操作）
sudo -u postgres psql
# 执行 SQL 命令（见指南第三步）

# 5. 配置 Redis（编辑配置文件）
vim /etc/redis/redis.conf
# 设置 requirepass（见指南第四步）
systemctl restart redis-server

# 6. 配置环境变量
cp deploy/env/allo.env.production.example /etc/allo/allo.env
vim /etc/allo/allo.env
# 修改所有 CHANGE_ME 的值

# 7. 安装依赖
sudo -u allo -i
cd /srv/allo/backend
source /etc/allo/allo.env
uv sync

cd /srv/allo/frontend
pnpm install

# 8. 运行数据库迁移
cd /srv/allo/backend
PYTHONPATH=. uv run alembic upgrade head

# 9. 构建前端
cd /srv/allo/frontend
pnpm build

# 10. 安装 systemd 服务
exit  # 退出 allo 用户
cp /srv/allo/deploy/systemd/*.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable allo-gateway allo-langgraph allo-frontend
systemctl start allo-gateway allo-langgraph allo-frontend

# 11. 配置 nginx
cp /srv/allo/deploy/nginx/allo-production.conf /etc/nginx/sites-available/allo
ln -s /etc/nginx/sites-available/allo /etc/nginx/sites-enabled/allo
rm /etc/nginx/sites-enabled/default  # 可选
nginx -t
systemctl reload nginx

# 12. 验证部署
curl http://localhost/health
curl -X POST http://localhost/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@allo.local","password":"Admin123!","display_name":"Admin"}'

# 13. 浏览器访问
# http://YOUR_SERVER_IP
```

## 关键配置项

### 必须修改的环境变量

在 `/etc/allo/allo.env` 中：

```bash
# 数据库密码（第三步设置的）
DATABASE_URL=postgresql+asyncpg://allo:YOUR_PG_PASSWORD@127.0.0.1:5432/allo
CHECKPOINT_POSTGRES_URI=postgresql://allo:YOUR_PG_PASSWORD@127.0.0.1:5432/allo

# Redis 密码（第四步设置的）
REDIS_URL=redis://:YOUR_REDIS_PASSWORD@127.0.0.1:6379/0

# 安全密钥（生成随机值）
SESSION_SECRET=$(python3 -c "import secrets; print(secrets.token_urlsafe(32))")
API_KEY_ENCRYPTION_SECRET=$(python3 -c "import secrets; print(secrets.token_urlsafe(32))")
BETTER_AUTH_SECRET=$(python3 -c "import secrets; print(secrets.token_urlsafe(32))")

# 平台模型 Key（至少配置一个）
OPENAI_API_KEY=sk-...
TAVILY_API_KEY=tvly-...

# 服务器 IP
ALLOWED_ORIGINS=http://YOUR_SERVER_IP
NEXT_PUBLIC_APP_URL=http://YOUR_SERVER_IP
NEXT_PUBLIC_GATEWAY_URL=http://YOUR_SERVER_IP/api
NEXT_PUBLIC_LANGGRAPH_URL=http://YOUR_SERVER_IP/api/langgraph
```

## 验证检查点

### 服务状态
```bash
systemctl status postgresql redis-server nginx
systemctl status allo-gateway allo-langgraph allo-frontend
```

### 端口监听
```bash
ss -tlnp | grep -E '(80|3000|8001|2024|5432|6379)'
```

### 日志查看
```bash
journalctl -u allo-gateway -f
journalctl -u allo-langgraph -f
journalctl -u allo-frontend -f
tail -f /var/log/nginx/allo-error.log
```

## 故障排查

### Gateway 启动失败
```bash
journalctl -u allo-gateway -n 100 --no-pager
# 常见问题：数据库连接失败、Redis 连接失败、端口占用
```

### LangGraph 启动失败
```bash
journalctl -u allo-langgraph -n 100 --no-pager
# 常见问题：config.yaml 找不到、checkpoint 连接失败
```

### Frontend 构建失败
```bash
cd /srv/allo/frontend
pnpm build
# 常见问题：BETTER_AUTH_SECRET 未设置、依赖未安装
```

### nginx 502 错误
```bash
# 检查上游服务
curl http://localhost:3000
curl http://localhost:8001/health
curl http://localhost:2024/health
```

## 安全建议

1. 修改所有默认密码为强密码
2. 配置防火墙只开放 80/443/22 端口
3. 定期更新系统：`apt update && apt upgrade`
4. 定期检查日志：`journalctl -p err -n 100`
5. 后续购买域名后配置 HTTPS
