# Allo 服务器环境准备指南

本文档用于在生产服务器上准备 Allo 运行环境。

## 服务器要求

- **操作系统**: Ubuntu 22.04 或 24.04 LTS
- **配置**: 16C / 32G / >=200GB SSD
- **网络**: 公网 IP（第一版用 HTTP，后续加域名和 HTTPS）
- **权限**: root 或 sudo 权限

---

## 第一步：基础软件安装

以 root 或 sudo 用户执行：

```bash
# 更新系统
apt update && apt upgrade -y

# 安装基础工具
apt install -y curl wget git vim build-essential

# 安装 PostgreSQL 16
apt install -y postgresql-16 postgresql-contrib-16

# 安装 Redis
apt install -y redis-server

# 安装 nginx
apt install -y nginx

# 安装 Python 3.12（Ubuntu 24.04 自带，22.04 需要 PPA）
apt install -y python3.12 python3.12-venv python3.12-dev

# 安装 uv（Python 包管理器）
curl -LsSf https://astral.sh/uv/install.sh | sh
source $HOME/.cargo/env

# 安装 Node.js 22
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

# 安装 pnpm
npm install -g pnpm

# 安装 Docker（用于 sandbox）
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker
```

## 第二步：创建应用用户和目录

```bash
# 创建应用用户（不允许登录）
useradd -r -m -s /bin/bash allo

# 创建应用目录
mkdir -p /srv/allo
chown allo:allo /srv/allo

# 创建数据目录
mkdir -p /var/lib/allo/users
chown -R allo:allo /var/lib/allo

# 创建日志目录
mkdir -p /var/log/allo
chown allo:allo /var/log/allo

# 创建备份目录
mkdir -p /backups
chown allo:allo /backups

# 创建配置目录
mkdir -p /etc/allo
chown allo:allo /etc/allo
```

## 第三步：配置 PostgreSQL

```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 在 psql 中执行：
CREATE USER allo WITH PASSWORD 'your-strong-password-here';
CREATE DATABASE allo OWNER allo;
\c allo
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pgcrypto";
\q

# 配置 PostgreSQL 允许本地连接
# 编辑 /etc/postgresql/16/main/pg_hba.conf
# 添加以下行（在 local all postgres peer 之后）：
local   allo            allo                                    md5
host    allo            allo            127.0.0.1/32            md5

# 重启 PostgreSQL
systemctl restart postgresql

# 验证连接
sudo -u allo psql -h 127.0.0.1 -U allo -d allo -c "SELECT 1"
# 输入密码后应该返回 1
```

---

## 第四步：配置 Redis

```bash
# 编辑 /etc/redis/redis.conf
# 找到以下行并修改：

# 1. 设置密码（取消注释并修改）
requirepass your-redis-password-here

# 2. 绑定到本地（默认已是，确认即可）
bind 127.0.0.1 ::1

# 3. 启用持久化（默认已启用，确认即可）
save 900 1
save 300 10
save 60 10000

# 重启 Redis
systemctl restart redis-server

# 验证连接
redis-cli -a your-redis-password-here ping
# 应该返回 PONG
```

## 第五步：部署应用代码

```bash
# 切换到 allo 用户
sudo -u allo -i

# 克隆代码（或从本地 rsync）
cd /srv/allo
git clone <your-repo-url> .

# 或者从本地同步：
# 在本地执行：
# rsync -avz --exclude='.git' --exclude='node_modules' --exclude='.next' \
#   /Users/steven/VaaT-Flow/ root@your-server-ip:/srv/allo/

# 安装后端依赖
cd /srv/allo/backend
uv sync

# 安装前端依赖
cd /srv/allo/frontend
pnpm install

# 构建前端（需要先配置环境变量）
# 稍后在配置环境变量后执行
```

## 第六步：配置环境变量

```bash
# 复制环境变量模板
sudo -u allo cp /srv/allo/deploy/env/allo.env.production.example /etc/allo/allo.env

# 编辑环境变量
sudo -u allo vim /etc/allo/allo.env
```

**必须修改的变量**（见下方 `allo.env.production.example` 文件）：
- `POSTGRES_PASSWORD` — 第三步设置的 PG 密码
- `REDIS_PASSWORD` — 第四步设置的 Redis 密码
- `SESSION_SECRET` — 生成随机 32 字符
- `API_KEY_ENCRYPTION_SECRET` — 生成随机 32 字符
- `OPENAI_API_KEY` / `TAVILY_API_KEY` 等 — 平台模型 Key

生成随机密钥：
```bash
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

## 第七步：初始化数据库

```bash
# 切换到 allo 用户
sudo -u allo -i

# 加载环境变量
source /etc/allo/allo.env

# 运行 Alembic 迁移
cd /srv/allo/backend
PYTHONPATH=. uv run alembic upgrade head

# 验证表已创建
psql -h 127.0.0.1 -U allo -d allo -c "\dt"
# 应该看到 22 张表
```

## 第八步：构建前端

```bash
# 切换到 allo 用户
sudo -u allo -i

# 加载环境变量
source /etc/allo/allo.env

# 构建前端
cd /srv/allo/frontend
pnpm build

# 验证构建产物
ls -la .next/standalone
```

---

## 第九步：安装 systemd 服务

```bash
# 复制 service 文件到 systemd 目录
sudo cp /srv/allo/deploy/systemd/*.service /etc/systemd/system/

# 重载 systemd
sudo systemctl daemon-reload

# 启用服务（开机自启）
sudo systemctl enable allo-gateway
sudo systemctl enable allo-langgraph
sudo systemctl enable allo-frontend

# 启动服务
sudo systemctl start allo-gateway
sudo systemctl start allo-langgraph
sudo systemctl start allo-frontend

# 检查状态
sudo systemctl status allo-gateway
sudo systemctl status allo-langgraph
sudo systemctl status allo-frontend

# 查看日志
sudo journalctl -u allo-gateway -f
sudo journalctl -u allo-langgraph -f
sudo journalctl -u allo-frontend -f
```

---

## 第十步：配置 nginx

```bash
# 复制 nginx 配置
sudo cp /srv/allo/deploy/nginx/allo.conf /etc/nginx/sites-available/allo

# 创建软链接
sudo ln -s /etc/nginx/sites-available/allo /etc/nginx/sites-enabled/allo

# 删除默认站点（可选）
sudo rm /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重载 nginx
sudo systemctl reload nginx
```

## 第十一步：验证部署

```bash
# 1. 检查所有服务运行状态
sudo systemctl status postgresql redis-server nginx allo-gateway allo-langgraph allo-frontend

# 2. 检查端口监听
ss -tlnp | grep -E '(80|3000|8001|2024|5432|6379)'

# 3. 测试 Gateway 健康检查
curl http://localhost:8001/health

# 4. 测试 nginx 反代
curl http://localhost/health

# 5. 测试注册用户
curl -X POST http://localhost/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@allo.local","password":"Admin123!","display_name":"Admin"}'

# 6. 测试登录
curl -c /tmp/cookies.txt -X POST http://localhost/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@allo.local","password":"Admin123!"}'

# 7. 测试获取用户信息
curl -b /tmp/cookies.txt http://localhost/api/users/me

# 8. 浏览器访问
# 打开浏览器访问 http://your-server-ip
# 应该看到登录页面
```

## 第十二步：配置备份

```bash
# 复制备份脚本
sudo cp /srv/allo/scripts/backup.sh /usr/local/bin/allo-backup
sudo chmod +x /usr/local/bin/allo-backup

# 编辑 crontab
sudo -u allo crontab -e

# 添加以下行（每天凌晨 2 点备份）
0 2 * * * /usr/local/bin/allo-backup >> /var/log/allo/backup.log 2>&1
```

## 故障排查

### Gateway 启动失败

```bash
# 查看详细日志
sudo journalctl -u allo-gateway -n 100 --no-pager

# 常见问题：
# 1. 数据库连接失败 → 检查 DATABASE_URL 和 PG 密码
# 2. Redis 连接失败 → 检查 REDIS_URL 和 Redis 密码
# 3. 端口被占用 → lsof -i :8001
```

### LangGraph 启动失败

```bash
# 查看日志
sudo journalctl -u allo-langgraph -n 100 --no-pager

# 常见问题：
# 1. config.yaml 找不到 → 检查 WorkingDirectory
# 2. checkpoint 连接失败 → 检查 CHECKPOINT_POSTGRES_URI
```

### Frontend 启动失败

```bash
# 查看日志
sudo journalctl -u allo-frontend -n 100 --no-pager

# 常见问题：
# 1. .next/standalone 不存在 → 重新 pnpm build
# 2. 端口被占用 → lsof -i :3000
```

### nginx 502 Bad Gateway

```bash
# 检查上游服务是否运行
curl http://localhost:3000
curl http://localhost:8001/health
curl http://localhost:2024/health

# 检查 nginx 错误日志
tail -f /var/log/nginx/error.log
```


## 安全建议

1. **修改默认密码**：PostgreSQL、Redis 密码必须强密码
2. **防火墙**：只开放 80/443/22 端口
3. **定期更新**：`apt update && apt upgrade`
4. **备份验证**：定期测试备份恢复流程
5. **日志监控**：定期检查错误日志
6. **HTTPS**：后续购买域名后配置 Let's Encrypt 证书
