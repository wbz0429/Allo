## Sprint 0：共同准备（两人一起，1-2 天）

在各自开干之前，必须先对齐接口契约。否则后面对接会反复返工。

### 0.1 服务器环境搭建（一起做）

```
在服务器上安装：
  ├── postgresql-16
  ├── redis
  ├── nginx
  ├── python 3.12 + uv
  ├── node 22 + pnpm
  ├── docker（仅 sandbox 用）
  └── git

创建：
  ├── 应用用户 allo
  ├── 应用目录 /srv/allo
  ├── 数据目录 /var/lib/allo/users
  ├── 日志目录 /var/log/allo
  └── PostgreSQL 数据库 allo + 扩展 uuid-ossp, pgcrypto
```

### 0.2 定义接口契约（一起做，最关键）

两人坐在一起，把以下契约写成代码文件（只写接口，不写实现）：

**契约 1：抽象接口（B 定义，A 实现）**

```python
# backend/packages/harness/deerflow/stores.py（新建）

class MemoryStore(ABC):
    async def get_memory(self, user_id: str) -> dict: ...
    async def save_memory(self, user_id: str, data: dict): ...
    async def get_facts(self, user_id: str, limit: int = 15) -> list[dict]: ...

class SoulStore(ABC):
    async def get_soul(self, user_id: str) -> str | None: ...

class SkillConfigStore(ABC):
    async def get_skill_toggles(self, user_id: str) -> dict[str, bool]: ...

class McpConfigStore(ABC):
    async def get_user_mcp_config(self, user_id: str) -> dict: ...

class ModelKeyResolver(ABC):
    async def resolve_key(self, run_id: str) -> tuple[str, str | None]: ...
    # returns (api_key, base_url)
```

**契约 2：UserContext（B 定义）**

```python
# backend/packages/harness/deerflow/context.py（新建）

@dataclass(frozen=True)
class UserContext:
    user_id: str
    org_id: str
    run_id: str | None = None
```

**契约 3：数据库 schema（A 定义，两人确认）**

把所有表的 CREATE TABLE 语句写出来，两人确认字段名和类型。

**契约 4：runtime.context 字段约定**

```
nginx 注入：
  x-user-id: str
  x-org-id: str

前端传入：
  thread_id: str
  run_id: str
  model_name: str
  agent_name: str
  thinking_enabled: bool
  is_plan_mode: bool
  subagent_enabled: bool
```

**契约 5：Redis key 约定**

```
session:{token}      → AuthContext JSON, TTL 5min    (A 写，A 读)
run:{run_id}:key     → API Key JSON, TTL 5min        (A 写，B 读)
rate_limit:{user_id} → 计数器, TTL 60s               (A 写，A 读)
```

**契约 6：文件路径约定**

```
/var/lib/allo/users/{user_id}/threads/{thread_id}/user-data/workspace/
/var/lib/allo/users/{user_id}/threads/{thread_id}/user-data/uploads/
/var/lib/allo/users/{user_id}/threads/{thread_id}/user-data/outputs/
/var/lib/allo/users/{user_id}/threads/{thread_id}/user-data/tmp/
/var/lib/allo/users/{user_id}/skills/custom/{skill_name}/
```

## 关键风险和应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 接口契约不完整，联调时大量返工 | Sprint 2 延期 | Sprint 0 多花时间，把接口写成代码而不是文档 |
| harness 改造影响本地开发 | 开发效率下降 | 所有改动加 `if user_id` 分支，无 user_id 回退到现有行为 |
| nginx auth_request 和 SSE 流式冲突 | 流式消息不通 | Sprint 1 就验证 auth_request + SSE 透传 |
| LangGraph checkpointer 切 PG 后性能下降 | 响应变慢 | 先在本地测试 PG checkpointer 性能 |
| sandbox 容器管理复杂度超预期 | Sprint 3 延期 | 第一版先用 LocalSandbox 兜底，Docker sandbox 作为增强 |
| 两人代码冲突 | 合并困难 | A 改 app/gateway，B 改 packages/harness，冲突面小 |
