## 四、为什么之前 `web_search` 不可用

根因不是模型本身，而是 marketplace 工具门控：

- 运行时 metadata 显示：
  - `resolved_managed_tools = ["bash", "web_search"]`
  - `resolved_installed_tools = []`
- 这表示 `web_search` 虽然在系统里定义了，但被标记为“受管工具”
- 当前组织未安装时，运行时会把它从可用工具列表中过滤掉

修复动作：

- 为当前组织安装 `tool-tavily-search`
- 补齐 `TAVILY_API_KEY`

修复后，运行时 metadata 已显示：

- `resolved_installed_tools = ["web_search"]`

## 七、仍需注意的问题

### 1. LangGraph 仍在 dev 启动模式

当前 `allo-langgraph` 仍使用开发模式启动，日志中会出现 `watchfiles` 相关噪音。虽然当前功能已可用，但长期生产环境建议改成更稳定的启动方式。

### 2. 运行时工具门控与 config 工具不是同一层

当前系统中存在两套概念：

- `config.yaml` 中声明的 runtime tools
- marketplace 安装态与 managed runtime tool gating

这意味着：

- 某些工具即使在 marketplace 中已安装，也不一定自动进入当前 runtime tool gating
- 某些工具即使未体现在 `resolved_installed_tools` 里，只要它是 runtime config tool，仍然可能可用

这部分建议后续统一梳理，避免“工具目录里看得到，但 agent 实际不一定可用”的认知混淆。

### 3. 注册接口有限流

注册接口当前有每 IP 每小时 5 次限流。测试时频繁创建账号会触发限制，这属于预期行为，不是线上 bug。
