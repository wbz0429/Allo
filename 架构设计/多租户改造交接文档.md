## 二、遗留问题 & 需要验证的事项

### 高优先级

1. **`make dev` 启动验证**
   - 需要先配置 `config.yaml`（已从 `config.example.yaml` 复制，但需要填入实际的模型 API key）
   - 验证 Gateway (8001) + LangGraph (2024) + Frontend (3000) 三个服务都能正常启动
   - 验证 SKIP_AUTH=1 模式下本地开发是否正常（向后兼容性）

2. **Auth 流程端到端验证**
   - 注册 → 登录 → session cookie → 访问受保护页面 → 登出
   - Redis session 缓存是否正常工作
   - `get_auth_context` 的 cookie → Redis → PG 三级查找链路

3. **Thread 生命周期验证**
   - 前端创建 thread → Gateway 写 PG → LangGraph 创建 thread → 消息流
   - Thread 列表查询、重命名、删除
   - thread_run 状态更新（running → finished/error）

4. **Store 注册 + Harness 消费验证**
   - Gateway 启动时 `register_store()` 是否成功
   - `make_lead_agent` 中 `get_store()` 是否能拿到实现
   - per-user memory/soul/skills 加载是否正确回退到文件模式

### 中优先级

5. **前端 Better Auth 残留清理**
   - 检查是否有其他文件仍然 import `better-auth` 相关模块
   - `frontend/package.json` 中 `better-auth` 依赖是否需要移除
   - 检查 `frontend/src/app/admin/layout.tsx` 等文件的 auth 引用

6. **nginx auth_request 联调**
   - `deploy/nginx/allo.conf` 目前是模板，需要在实际部署时验证
   - `/api/auth/check` 端点返回的 X-User-Id / X-Org-Id header 是否被 LangGraph 正确接收

7. **Alembic 迁移健壮性**
   - 在全新数据库上 `alembic upgrade head` 是否幂等
   - 后续新增迁移时 revision chain 是否正确（当前 001 → 002 → 004，缺 003）

### 低优先级

8. **测试覆盖**
   - workers 创建了一些测试文件（test_paths_user_dirs.py, test_mcp_cache.py 等），需要验证是否能通过
   - Auth API 缺少完整的集成测试
   - Thread CRUD API 缺少测试

9. **代码质量**
   - 运行 `ruff check` 检查所有新增/修改文件
   - 部分 worker 生成的代码可能有 LSP 警告（worktree 环境无 venv 导致的 import 误报 vs 真实问题）

## 三、架构关键约定

### Cookie 约定
```
名称: session_token (不再是 better-auth.session_token)
格式: session_token={token}; HttpOnly; Path=/; SameSite=Lax; Max-Age=604800
```

### Redis Key 约定
```
session:{token}       → AuthContext JSON, TTL 300s
run:{run_id}:key      → {api_key, base_url} JSON, TTL 300s
```

### 向后兼容规则
- 所有 Harness 层改动: user_id=None 时保持原有文件行为
- SKIP_AUTH=1 环境变量: 本地开发跳过认证，使用 dev-user-000
- `make dev` 不需要 PostgreSQL/Redis 也能跑（SKIP_AUTH 模式）

### 文件所有权边界
```
A 团队: backend/app/gateway/, backend/alembic/, frontend/, deploy/
B 团队: backend/packages/harness/deerflow/
共享:   stores.py, context.py, store_registry.py (接口契约)
```
